## 命名规范

### 文件命名

- **组件文件**: PascalCase，如 `PortfolioList.vue`
- **工具函数**: camelCase，如 `formatDate.ts`
- **类型定义**: PascalCase，如 `index.ts` (导出多个类型)
- **样式文件**: kebab-case，如 `global-styles.css`

### 组件命名

- 使用多个单词，如 `PortfolioList.vue`
- 与文件名保持一致
- 使用语义化的名称

### 变量命名

```typescript
// 布尔值使用 is/has/should 等前缀
const isLoading = ref(false)
const hasError = ref(false)
const shouldRefresh = ref(true)

// 数组使用复数形式

// 对象使用单数形式
const currentUser = ref<User | null>(null)
const selectedPortfolio = ref<Portfolio | null>(null)

// 函数使用动词开头
const fetchPortfolios = async () => {}
const handleCreate = () => {}
```

## 状态管理规范

### Pinia Store

```typescript
// store/modules/portfolio.ts
import { defineStore } from 'pinia'
import { ref } from 'vue'
import { getPortfolios } from '@/api/services/portfolio'
import type { Portfolio } from '@/types'

export const usePortfolioStore = defineStore('portfolio', () => {
  const portfolios = ref<Portfolio[]>([])
  const loading = ref(false)
  
  const fetchPortfolios = async () => {
    loading.value = true
    try {
      const response = await getPortfolios()
      portfolios.value = response
    } catch (error) {
      console.error(error)
    } finally {
      loading.value = false
    }
  }
  
  return {
    portfolios,
    loading,
    fetchPortfolios
  }
})
```

## 提交规范

### Git Commit Message

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**：
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具链相关

**示例**：
```
feat(portfolio): 添加投资组合默认组合功能

- 添加 is_default 字段
- 实现默认组合自动设置逻辑
- 更新投资组合列表显示默认标签

Closes #123
```

## 性能优化

### 组件优化

- 使用 `v-show` 替代 `v-if` 频繁切换的元素
- 列表渲染使用唯一的 `key`
- 避免在内联函数中创建新对象

```vue
<!-- 好的做法 -->
<el-button @click="handleClick(item.id)">{{ item.name }}</el-button>

<!-- 避免 -->
<el-button @click="() => handleClick(item.id)">{{ item.name }}</el-button>
```

### 数据优化

- 大数据量列表使用虚拟滚动
- 图片使用懒加载
- API 请求使用防抖/节流
